# Attendance Record — AGENTS.md

## Project structure

All source is in `src/`. Docker-compose with 5 services (app, nginx:8080, db:3307, phpmyadmin:8081, vite:5173).

```
src/
├── app/
│   ├── Filament/Resources/{Attendances,Events,Headquarters,Positions,Reasons}/
│   ├── Http/Controllers/{PublicAttendance,Report}Controller.php
│   ├── Models/{Event,Attendance,User,Position,Headquarter,Reason}.php
│   ├── Providers/Filament/AdminPanelProvider.php
│   └── Services/{Attendance,Event,QrCode,Report}Service.php
├── resources/
│   ├── css/
│   │   ├── app.css                    ─ public pages theme (navy/gold/warm)
│   │   └── filament/admin/theme.css   ─ Filament admin theme overrides
│   ├── js/app.js                      ─ signature_pad init
│   └── views/
│       ├── public/event-register.blade.php  ─ public registration form
│       ├── welcome.blade.php                ─ landing page
│       └── filament/resources/{attendances,events}/
├── routes/web.php
└── vite.config.js
```

## Key setup

```bash
docker compose up -d                             # start all services
docker compose run --rm app php artisan migrate   # run migrations
docker compose run --rm app php artisan db:seed   # seed (admin@example.com / admin123)
```

Asset building (runs inside `vite` container):
```bash
docker compose run --rm vite npm run build
# Or from outside, clean with: docker compose run --rm -u root vite rm -rf /var/www/public/build
```

## Two separate CSS entry points

Custom colors (`navy-*`, `gold-*`, `warm-*`) are defined via `@theme` in **both** CSS files independently — Tailwind v4 processes each entry separately, so colors in `app.css` are NOT available in `filament/admin/theme.css`. Always redefine or use raw hex values.

## Admin panel customization

Edit `app/Providers/Filament/AdminPanelProvider.php`:
- `->brandName('Asistencia')`, `->font('Instrument Sans')`, `->colors(['primary' => Color::Amber])`
- `->viteTheme('resources/css/filament/admin/theme.css')` registers the custom Filament theme
- Login page styled via CSS overrides in `resources/css/filament/admin/theme.css`

Custom CSS in `resources/css/filament/admin/theme.css` targets Filament classes (`.fi-sidebar`, `.fi-topbar`, `.fi-btn`, `.fi-input-wrp`, `.fi-card`, `.fi-ta-ctn`, `.fi-modal-window`, etc.).

## Fonts

Loaded via `laravel-vite-plugin`'s Bunny CDN integration (`@fonts` directive in Blade):
- **Body**: Instrument Sans (400, 500, 600)
- **Headings**: DM Serif Display (400) — used via `font-serif` class

## Models & relationships

```
Event → directedBy (User)  |  Event → hasMany(Attendance)
Attendance → belongsTo(Event, Position, Headquarter)
User → hasMany(Event, foreignKey: directed_by_id)
```

## Routes (web.php)

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/` | Welcome page |
| GET/POST | `/evento/{slug}` | Public registration |
| GET | `/admin/reportes/asistencias?event_id=` | CSV export (auth) |

## Key conventions

- All text in Spanish (UI labels, validation messages)
- `is_active` boolean on Position, Headquarter, Reason for soft-disable
- Attendance `signature` column stores a data URI string (from signature_pad canvas)
- Unique constraint: one registration per `(event_id, id_number)`
- `slug` on Event is auto-generated by `EventService::generateSlug()`
